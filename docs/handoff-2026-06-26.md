## 一、项目概述

ComfyUI 自定义节点包 `comfyui_awp_rp`——AI 驱动的交互式叙事/RP 工作流。

核心链路:
```
用户输入 → AWPRoundRouter(路由决策) → AWPRoundPreparer(世界书+预算)
→ AWPMemoryRead(记忆门控) → AWPSubAgentOrchestrator(子Agent+结构化查询)
→ AWPMainAgent(Writer+安全屏障) → AWPQualityGate → AWPMemoryWrite → 输出
```

## 四、P3B 真实 API 发现的关键问题

### A. Curator 写入失败 (15次触发,14次失败)

**根因**: `rp-memory-curator` 调用 DeepSeek 时返回空字符串。独立 smoke 复现:
```python
result = _run_sub_agent(profile_id='rp-memory-curator',
    task='Extract structured memories...',
    context='...real RP context...')
# result = '' (empty string)
```

对比:
- curator "say hello" (最小 prompt) → 成功
- `rp-critic` (`response_format='json_object'`) → 成功
- curator 完整 RP context → 空字符串或 `[]`

**不是**超时、profile 配置、schema 问题。**是模型行为**:DeepSeek 面对真实 RP 上下文 + curator 的 JSON 提取指令,返回空响应。

**修复方向 (P4A)**: 改 curator profile——加 `response_format='json_object'` 强约束,或改用更简单的提取 prompt,或换模型。

### B. Writer 质量三大缺陷

1. **角色名漂移**: 26回合中出现 8 个不同的角色名(周语晴→翠英→苏杏儿→秀兰→青荷→宋招娣→宋怀生→栓子哥)。第一次漂移在 T02 即发生。根因:世界书条目过多(每轮 13-19 条,~2500 token),角色锚定被稀释。

2. **回复过短**: 正文平均 295 字,最大 477 字,**0/26 回合达到 800 字**。`rp-writer` profile 无具体字数硬约束(max_tokens=2048 够但未利用)。`<options>` 块平均消耗 170 字。

3. **剧情脱轨**: 公公-儿媳核心设定在 T8 后漂变为"知青下乡恋爱"。原因:Writer prompt 中角色身份、关系、场景锚定不够强;用户输入风格偏文艺恋爱叙事,未保持老农口吻。

**修复方向 (P4B)**: 强化 `rp-writer` profile——(a) 加硬字数要求(≥800字),(b) 在 prompt 顶部锚定角色名+身份+关系(如"你是老马,桃花村农户;儿媳=周语晴,仅此一人"),(c) 减 worldbook 条目或标优先级,(d) `<options>` 不占正式输出 token 预算。
